# LegalDoc AI Eval — 結構化法律文件的 Prompt 策略比較框架

本專案是一個 AI 評估（Eval）框架，設計目的是對大量結構化文件進行系統性的 prompt 策略比較，並輸出量化報告。
**示範應用**：台灣法院外遇相關民事判決書。輸入 PDF 判決書，自動萃取案件結構，並以 5 種不同的 prompt 策略分析每份文件，產生視覺化 HTML 比較報告。

## 這個框架的其他用途

判決書分析只是一個示範用途。這套框架的核心邏輯——讀入非結構化 PDF、以多種 prompt 策略萃取結構化資訊、量化比較各策略的表現——可以直接套用到以下場景：

| 應用場景 | 文件類型 | 萃取目標 |
|---|---|---|
| ESG / 永續報告書分析 | 上市公司 CSR 報告 PDF | 碳排放目標、治理結構、揭露完整度 |
| 法遵自動化 | 合約、內控文件、監理函令 | 義務條款、違約條件、合規缺口 |
| 盡職調查輔助 | 財報、法院公告、股東會議事錄 | 風險事項、關係人交易、訴訟揭露 |
| 政策法規追蹤 | 行政命令、立法草案 | 修訂重點、影響範圍、施行日期 |
| 醫療或藥品審查 | 藥品仿單、臨床試驗報告 | 適應症、副作用、禁忌事項 |

專案重點：

- **Prompt Engineering**：設計並系統比較不同問法（Zero-Shot、Few-Shot、Chain-of-Thought、格式模板）對同一語料的輸出差異
- **LLM Evaluation**：建立可量化的評估指標（token 用量、輸出長度、結構萃取成功率）
- **Document Pipeline**：從 PDF 解析、欄位萃取到結構化儲存的完整流程
- **Reproducibility**：同一份文件以多種策略跑出可比較的結果，輸出 JSON、Markdown、HTML 三種格式

## 專案架構

```
你外遇!Eval/
├── main.py          # 主程式：PDF 解析、API 呼叫、結果儲存
├── prompts.py       # 5 種 prompt 策略定義 + 結構化萃取 prompt
├── report.py        # HTML 視覺化報告產生器
├── requirements.txt # 套件清單
├── .env             # OpenAI API Key（勿提交至版本控制）
├── verdicts/        # 輸入文件（PDF 或 TXT）
└── results/         # 輸出報告（HTML、JSON、Markdown）
```

## 5 種 Prompt 策略

本框架同時執行以下五種策略，並在報告中並排比較輸出品質與 token 成本：

| 策略 | 設計邏輯 | 適用情境 |
|---|---|---|
| **白話版** | 要求模型以非專業讀者為對象輸出，附證據強度星評 | 使用者研究、非技術利害關係人溝通 |
| **Zero-Shot** | 直接提問，不提供範例或格式限制 | 快速探索、評估模型基礎理解能力 |
| **格式模板** | 要求模型依固定欄位回答 | 需要下游程式解析的結構化輸出 |
| **Few-Shot** | 提供 2 個標注範例後再提問 | 要求輸出一致性、格式收斂 |
| **逐步推理** | 要求模型按步驟拆解問題後再下結論 | 複雜推理、提升準確率、減少幻覺 |

除上述 5 種策略外，每份文件還會執行一次獨立的**結構萃取呼叫**，以嚴格格式輸出以下欄位，作為跨文件統計的基礎：

- 主要結論（是/否/不明）
- 量化數字（本示範為損害賠償金額）
- 事證類型標籤

## 快速開始

### 環境需求

- Python 3.10 以上
- OpenAI API Key

### 安裝

```bash
# 建立虛擬環境
python -m venv venv

# 啟動（Windows）
venv\Scripts\activate

# 啟動（Mac / Linux）
source venv/bin/activate

# 安裝套件
pip install -r requirements.txt
```

### 設定 API Key

編輯 `.env` 檔案：

```
OPENAI_API_KEY=sk-proj-你的金鑰
```

### 放入文件

將 PDF 或 TXT 文件放入 `verdicts/` 資料夾。程式會自動批次處理所有檔案。

`verdicts/` 內已附一份虛構範例判決書（`範例判決書_虛構資料.txt`），可直接執行以驗證環境設定是否正確，無需先準備自己的文件。

### 執行

```bash
python main.py
```

每份文件執行 5 種策略加 1 次結構萃取，完成後在 `results/` 產生報告。

## 輸出報告

`results/` 資料夾會產生三個同名檔案：

| 格式 | 用途 |
|---|---|
| `.html` | 視覺化報告，建議用瀏覽器開啟 |
| `.json` | 完整原始資料，供下游程式使用 |
| `.md` | 純文字版本，適合版本控制或文件嵌入 |

HTML 報告包含：

- **總覽統計**：結論分布（成立/不成立/不明）、量化數字彙整
- **事證有效性分析**：各證據類型的出現頻率與勝訴率橫向比較（含視覺化長條圖）
- **賠償金額分布圖**：各文件量化結果的視覺化比較
- **策略效能比較表**：5 種策略的平均 token 用量與回應時間
- **逐文件分析**：5 個 tab 並排顯示各策略輸出，支援依結論篩選

## 套用至其他文件類型

1. 修改 `prompts.py` 中的 5 種策略 prompt，描述你的分析目標
2. 修改 `evidence_extraction` 函式，定義你要萃取的結構化欄位
3. 將新文件放入 `verdicts/`，執行 `python main.py`

框架本身不針對特定文件類型，任何需要批次分析的非結構化文件都可以套用。
