## 1. 這個專案在做什麼

這個專案讀取你放入的 PDF 或文字檔，透過 OpenAI 的語言模型，以五種不同的提問策略分析每份文件，最終產生一份視覺化的 HTML 比較報告。

**五種策略的設計邏輯：**

| 策略 | 目的 |
|---|---|
| 白話版 | 將法律文件轉譯為一般人可理解的敘述 |
| Zero-Shot | 直接提問，測試模型的基礎理解能力 |
| 格式模板 | 要求模型依固定欄位輸出，便於程式解析 |
| Few-Shot | 提供範例後再提問，改善輸出一致性 |
| 逐步推理 | 要求模型按步驟分析，提升複雜推理準確率 |

每份文件還會額外執行一次結構化萃取，自動判斷「是否成立」、「賠償金額」、「出現的證據類型」，作為跨文件統計的依據。

## 3. 安裝 Python

本專案需要 Python 3.10 以上版本。

### 確認電腦上是否已有 Python

開啟終端機（Windows 使用命令提示字元或 PowerShell，Mac 使用 Terminal），輸入：

```
python --version
```

若顯示 `Python 3.10.x` 或更高版本，可跳過本節。若顯示「找不到指令」或版本低於 3.10，請繼續閱讀。

### Windows 安裝步驟

1. 前往 [https://www.python.org/downloads/](https://www.python.org/downloads/)
2. 點選「Download Python 3.x.x」（選最新穩定版）
3. 執行下載的安裝程式
4. **重要**：安裝畫面最下方有一個選項「Add Python to PATH」，務必勾選，否則後續步驟會無法執行 Python
5. 點選「Install Now」
6. 安裝完成後，重新開啟終端機，再次執行 `python --version` 確認安裝成功

### Mac 安裝步驟

1. 前往 [https://www.python.org/downloads/](https://www.python.org/downloads/)
2. 下載 macOS 版本的安裝程式並執行
3. 按照指示完成安裝
4. 重新開啟 Terminal，執行 `python3 --version` 確認

Mac 使用者在後續步驟中，若 `python` 無法執行，請嘗試 `python3`。

## 4. 取得 OpenAI API 金鑰


### 取得步驟

1. 前往 [https://platform.openai.com/](https://platform.openai.com/) 並登入或註冊帳號
2. 點選右上角頭像，選擇「API Keys」
3. 點選「Create new secret key」
4. 為金鑰取一個名稱（例如「eval-project」），點選「Create secret key」
5. **立刻複製這組金鑰**，格式為 `sk-proj-...`。這組金鑰只會顯示一次，關閉視窗後無法再查看

### 費用說明

使用 `gpt-4o-mini` 模型的費用非常低。以本示範專案為例，分析 10 份判決書（5 種策略加 1 次結構萃取）約需 **$0.09 美元**（約新台幣 3 元）。

首次使用 OpenAI API 帳號通常會附贈一定額度的免費試用額度，足以完整跑完本專案。

### 注意事項

- 不要將 API 金鑰分享給他人，也不要提交至 GitHub 或公開平台
- 本專案已將 `.env` 加入 `.gitignore`，你的金鑰不會被意外上傳

## 6. 建立虛擬環境並安裝套件


### 建立虛擬環境

在專案資料夾中執行：

```bash
python -m venv venv
```

這會在專案資料夾內建立一個名為 `venv` 的資料夾，裡面包含獨立的 Python 環境。

### 啟動虛擬環境

每次使用這個專案前，都需要先啟動虛擬環境。

**Windows（命令提示字元）：**
```
venv\Scripts\activate
```

**Windows（PowerShell）：**
```
venv\Scripts\Activate.ps1
```

若 PowerShell 顯示「執行原則」錯誤，請先執行：
```
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```
然後再試一次啟動指令。

**Mac / Linux：**
```
source venv/bin/activate
```

啟動成功後，終端機提示符前方會出現 `(venv)` 字樣，例如：
```
(venv) C:\Users\diana\Desktop\你外遇!Eval>
```

這表示虛擬環境已啟動，之後安裝的套件都會裝在這個獨立空間內。

### 安裝套件

虛擬環境啟動後，執行：

```bash
pip install -r requirements.txt
```

程式會自動安裝所有必要的套件，包含：
- `openai`：與 OpenAI API 溝通
- `pymupdf`：解析 PDF 文字（支援中文字型）
- `python-dotenv`：讀取 .env 設定檔

安裝完成後，終端機會顯示 `Successfully installed ...`。

## 7. 設定 API 金鑰

專案根目錄有一個 `.env` 檔案，用於存放 API 金鑰。

### 編輯 .env 檔案

用任意文字編輯器開啟 `.env`（記事本、VS Code 皆可），內容如下：

```
OPENAI_API_KEY=sk-proj-你的金鑰
```

將 `sk-proj-你的金鑰` 替換為你在第 4 節取得的實際金鑰，例如：

```
OPENAI_API_KEY=sk-proj-abc123xyz456...
```

等號前後不要加空格，不要加引號。

**找不到 .env 檔案？**

`.env` 是以句點開頭的隱藏檔案，某些作業系統預設不顯示。

- Windows：在檔案總管中，選擇「檢視」→「顯示」→「隱藏的項目」
- Mac：在 Finder 中按 `Command + Shift + .` 顯示隱藏檔案

若專案中沒有 `.env` 檔案，請自行建立一個純文字檔，命名為 `.env`（注意：只有副檔名，沒有主檔名），內容填入上述格式。

## 8. 放入文件並執行

### 放入文件

`verdicts/` 資料夾內已附一份虛構範例判決書（`範例判決書_虛構資料.txt`）。第一次執行時不需要準備自己的文件，可直接執行程式，確認環境設定無誤後再替換成你的文件。

若要分析自己的文件，將其放入 `verdicts/` 資料夾即可。支援兩種格式：

- **PDF**：程式會自動提取文字內容。注意：掃描圖片型 PDF（非文字型）無法解析，程式會顯示警告並跳過
- **TXT**：純文字檔，需以 UTF-8 編碼儲存

可以同時放入多份文件，程式會逐一處理。

確認虛擬環境已啟動（提示符前有 `(venv)`），然後執行：

## 9. 閱讀輸出報告

執行完成後，`results/` 資料夾會產生三個同名檔案（時間戳不同）：

| 檔案 | 說明 |
|---|---|
| `外遇分析報告_時間戳.html` | 視覺化報告，建議使用此檔案 |
| `外遇分析報告_時間戳.json` | 完整原始資料，供程式讀取使用 |
| `外遇分析報告_時間戳.md` | 純文字版本，可在任何文字編輯器開啟 |

### 開啟 HTML 報告

直接用瀏覽器（Chrome、Edge、Firefox 皆可）開啟 `.html` 檔案。

報告包含以下區塊：

**總覽統計**
頁面頂部顯示本批文件的基本統計：判決書總數、各結論類型的數量分布。

**判決結果一覽**
列表顯示每份文件的認定結果、賠償金額、主要事證類型，點擊文件名稱可跳轉至詳細分析。

**證據要件有效性分析**
跨文件統計各類型證據的出現頻率與勝訴率，以長條圖呈現。

**損害賠償金額分布**
將各文件的賠償金額由高至低排序，以橫條圖比較。

**策略效能比較**
比較五種策略的平均 token 用量與回應時間，量化評估各策略的成本效益。

**各判決書詳細分析**
每份文件以卡片形式呈現，切換五個分頁可查看各策略的輸出內容及 token 用量。

頁面頂部有篩選按鈕，可依「外遇成立」、「外遇不成立」、「證據不明」篩選顯示的文件。

## 10. 關掉後再次使用

### 再次執行的流程

每次重新開啟終端機後，需要重新啟動虛擬環境：

**Windows：**
```
cd C:\Users\你的使用者名稱\Desktop\你外遇!Eval
venv\Scripts\activate
python main.py
```

**Mac / Linux：**
```
cd /路徑/到/你外遇-Eval
source venv/bin/activate
python main.py
```

### 舊結果不會被覆蓋

每次執行會以當下的時間戳命名新檔案，例如：

```
results/
├── 外遇分析報告_20260423_140000.html
├── 外遇分析報告_20260423_140000.json
├── 外遇分析報告_20260423_140000.md
├── 外遇分析報告_20260424_093015.html   ← 第二次執行
├── 外遇分析報告_20260424_093015.json
└── 外遇分析報告_20260424_093015.md
```

舊的結果會完整保留，不會被新執行覆蓋。

### 新增文件後重新執行

若要新增文件分析，直接將新的 PDF 或 TXT 放入 `verdicts/` 資料夾，再次執行 `python main.py` 即可。

注意：每次執行會處理 `verdicts/` 內所有文件。若只想分析新增的文件，可先將舊文件移出 `verdicts/` 資料夾，分析完成後再移回。

## 11. 常見問題與錯誤排除

### `python` 或 `pip` 顯示「找不到指令」

**原因**：Python 未加入系統 PATH，或使用了錯誤的指令名稱。

**解決方式**：
- Windows：重新安裝 Python，確認勾選「Add Python to PATH」選項，安裝完後重新開啟終端機
- Mac：嘗試改用 `python3` 和 `pip3`

---

### `EnvironmentError: 請設定環境變數 OPENAI_API_KEY`

**原因**：程式找不到 API 金鑰。

**可能的情況：**
1. `.env` 檔案不存在 → 請按第 7 節建立
2. `.env` 檔案格式有誤 → 確認等號兩側無空格、無引號、無多餘符號
3. 虛擬環境未啟動就執行 → 先執行啟動指令再 `python main.py`

---

### `ModuleNotFoundError: No module named 'openai'`（或其他套件名稱）

**原因**：套件未安裝，或虛擬環境未啟動。

**解決方式**：
1. 確認終端機提示符前有 `(venv)` 字樣
2. 若沒有，先啟動虛擬環境，再執行 `pip install -r requirements.txt`

---

### `警告：XXX.pdf 無法解析文字（可能是掃描圖片 PDF）`

**原因**：該 PDF 是由掃描儀掃描後儲存的圖片，而非含有可選取文字的數位 PDF。`pymupdf` 無法從純圖片中提取文字。

**解決方式**：
- 使用 OCR 軟體（如 Adobe Acrobat、ABBYY FineReader）將圖片 PDF 轉換為文字 PDF，再放入 `verdicts/`
- 或手動將文件內容貼入 `.txt` 檔後放入 `verdicts/`

---

### `openai.AuthenticationError`

**原因**：API 金鑰無效或已過期。

**解決方式**：
1. 前往 [https://platform.openai.com/api-keys](https://platform.openai.com/api-keys) 確認金鑰狀態
2. 若金鑰已失效，建立新金鑰後更新 `.env` 檔案

---

### `openai.RateLimitError`

**原因**：短時間內送出太多 API 請求，或帳號額度已用盡。

**解決方式**：
- 若是額度問題：前往 OpenAI 帳號頁面確認餘額，必要時充值
- 若是速率限制（新帳號初期有較低的呼叫上限）：等待一分鐘後再重試

---

### `openai.APIConnectionError`

**原因**：網路連線問題，或 OpenAI 服務暫時中斷。

**解決方式**：
1. 確認網路連線正常
2. 前往 [https://status.openai.com/](https://status.openai.com/) 確認服務狀態
3. 若服務正常，稍後再試

---

### HTML 報告開啟後顯示亂碼

**原因**：瀏覽器未正確識別 UTF-8 編碼。

**解決方式**：
- Chrome / Edge：右鍵點擊頁面 → 「編碼」→ 選擇「UTF-8」
- 通常直接開啟不會有此問題，若發生請確認瀏覽器為最新版本

---

### 結果資料夾中有多份舊報告，不知道哪個是最新的

檔名中的時間戳格式為 `YYYYMMDD_HHMMSS`，數字最大的即為最新的執行結果。

## 12. 套用到其他文件類型

本框架的分析邏輯不限於外遇判決書，可以套用到任何需要批次分析的非結構化文件。

### 修改步驟

**步驟一：修改 `prompts.py`**

將五種策略函式中的提問內容，替換為符合你的分析目標的問法。以 ESG 報告書分析為例：

```python
def zero_shot(verdict_text: str) -> str:
    return f"""請分析以下 ESG 永續報告書，判斷該公司的碳排放目標是否具體可量化，並說明理由。

報告書內容：
{verdict_text}
"""
```

**步驟二：修改 `evidence_extraction` 函式**

調整結構化萃取的欄位，定義你要跨文件統計的項目：

```python
def evidence_extraction(verdict_text: str) -> str:
    return f"""請分析以下報告書，以嚴格格式輸出下列資訊，不要加任何額外說明。

碳中和目標年度: [純數字，無則填0]
科學基礎目標(SBT)認證: [是/否/不明]
揭露框架: [從以下選出有出現的，以逗號分隔: GRI, SASB, TCFD, CDP, 無]

報告書內容：
{verdict_text}
"""
```

**步驟三：調整 `report.py` 的顯示邏輯**

修改 `decision_badge()` 函式中的標籤文字，以及 `EVIDENCE_ORDER` 列表中的事證類型名稱，使其對應你的新欄位。

**步驟四：放入新文件並執行**

將新的 PDF 或 TXT 文件放入 `verdicts/` 資料夾，執行 `python main.py`，框架的其餘邏輯（API 呼叫、報告產生、統計圖表）維持不變，無需修改。
